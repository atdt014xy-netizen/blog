# 네이버 블로그 문체 스타일 가이드

> 이 문서는 필자(류수인)의 블로그 글 3편(AI 기술 / 여행 / 전자제품 리뷰)을 분석해 **말투와 문체 패턴만** 정리한 가이드입니다.
> 주제나 소재는 매번 달라져도 되지만, 아래 규칙을 따르면 "이 사람이 쓴 글"처럼 느껴지는 톤을 재현할 수 있습니다.
> 다른 글을 쓸 때 이 문서 하나만 참고해도 동일한 톤을 낼 수 있도록 규칙 + 실제 예문을 함께 제시합니다.

---

## 1. 기본 톤 요약

- **격식체(합니다/습니다)를 기본으로 쓰되, 딱딱하지 않게 곳곳에 "~는데요/~인데요" 같은 부드러운 종결을 섞는다.** 즉, 뼈대는 하십시오체, 숨결은 해요체.
- 전문가처럼 가르치는 투가 아니라 **"경험을 먼저 겪고, 그걸 솔직하게 나눠주는 친구"** 같은 태도.
- 확신은 강하게 말하되, 동시에 자신의 한계나 처음의 무덤덤함도 숨기지 않는다. → **정직함이 신뢰를 만든다는 전제**가 문체 전체에 깔려 있음.
- 독자를 계속 대화 상대로 소환한다 ("~라고 물으신다면", "~고민 중이시라면").

---

## 2. 문장 끝맺음 패턴

| 유형 | 예시 | 용도 |
|---|---|---|
| `~습니다 / ~합니다` | "저도 처음 ChatGPT를 써봤을 때 그렇게까지 감동받지 않았습니다." | 기본 서술, 단정적 진술 |
| `~는데요 / ~인데요` | "그런데 정작 중요한 변화는 훨씬 조용한 곳에서 일어나고 있는데요." | 문장을 마치되 여운을 남기고 다음 내용으로 자연스럽게 연결. **이 글의 시그니처 종결어미.** |
| `~것입니다` | "그것이 앞으로 10년을 준비하는 가장 확실한 방법입니다." | 문단이나 글 전체를 매듭짓는 선언적 마무리 |
| 짧은 단문 | "틀린 말이 아니었습니다. 당시에는요." | 임팩트를 위해 일부러 문장을 짧게 끊음 |
| 의문형 자문자답 | "왜 샀냐면", "우리는 어떻게 하면 좋을까요" | 소제목이나 문단 서두에서 질문을 던진 뒤 이어서 답함 |

**규칙**: 한 문단 안에 `~습니다`체 문장 2~3개가 이어지면, 그다음엔 반드시 `~는데요`나 `~인데요`로 한 번 숨을 고르게 한다. 이 리듬이 무너지면 톤이 딱딱해진다.

## 3. 자주 쓰는 표현 / 즐겨 쓰는 어휘

**문을 여는 표현**
- "솔직히 말씀드리면 / 솔직히"
- "저는 ~하는 편입니다" (자기 성향을 먼저 밝히기)
- "먼저 말씀드리고 싶은 것이 있습니다"

**전환/연결 표현**
- "그런데" (역접, 매우 자주 사용 — 딱딱한 "하지만"보다 구어적인 "그런데"를 선호)
- "그래서" (인과 연결, 다음 행동으로 넘어갈 때)
- "물론 ~라고 물으신다면" (예상 반론을 먼저 인정하고 넘어가는 패턴)

**강조/단정 표현**
- "확실히 다릅니다"
- "~것이 사실인데요" (인정 + 완곡)
- "무리가 없습니다" (긍정 평가를 단정할 때)
- "충분히 값어치를 합니다"

**독자에게 말 거는 표현**
- "~고민 중이시라면"
- "~하시길 권해 드립니다"
- "말씀드리자면", "~라고 물으신다면"

**부사/필러**
- 그냥, 정말, 확실히, 솔직히, 그냥저냥이 아니라 "그냥"을 자주 씀 (예: "그냥 동네였습니다", "그냥 거기 서 있었습니다") → 꾸미지 않은 담백함을 표현할 때 "그냥"을 반복 사용.

**대비 구조 (A가 아니라 B)**
- "제가 AI를 쓰는 방식이 달라진 게 아니라 — AI 없이 일하는 게 어색해진 것입니다."
- "XM5는 소리를 '줄여주는' 느낌이었는데요. XM6는 배경 자체가 사라지는 느낌입니다."
→ 두 대상을 대구로 놓고 차이를 부각하는 문장 구조를 즐겨 씀. 종종 **em dash(—)** 로 뒷부분을 강조.

---

## 4. 문장 호흡과 리듬

- **문단이 짧다.** 한 문단은 보통 1~3문장. 긴 설명도 문단을 잘게 쪼개서 여백을 준다.
- **한 문장짜리 문단으로 방점을 찍는다.** 예: "그래서 이 리뷰는 6개월간 매일 쓴 사람의 이야기입니다." 처럼 짧은 선언 문장을 독립 문단으로 배치해 강조.
- 긴 문장(정보 전달) → 짧은 문장(정서적 방점)의 순서로 리듬을 탄다.
- 대시(—)를 문장 중간에 넣어 숨 고르기/반전을 준다: "이 모든 것이 — 시간이 지날수록 — 격차로 이어집니다" 식의 삽입.
- 따옴표(" ")로 실제 생각이나 대사를 인용해 현장감을 준다: "오, 글을 꽤 잘 쓰네", "너무 이른 거 아니에요?"

## 5. 글의 구조 패턴

### 5-1. 제목 블록
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
[카테고리] 핵심 제목 — 부제 또는 한 줄 요약
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
- 제목은 `[카테고리] 메인 제목 — 보충 설명` 형태. 대시 뒤에 좀 더 감성적이거나 구체적인 문구를 덧붙임.
  (예: "GPT 이후의 세계 — 우리는 지금 어디쯤 와 있을까", "교토에서 3일 — 계획 없이 걷다가 길을 잃은 이야기")

### 5-2. 도입부 패턴
- 항상 **개인적 고백/솔직한 인상**으로 시작한다. 정보나 정의로 시작하지 않는다.
- "솔직히", "저는 ~하는 편입니다" 로 첫 문장을 여는 경우가 많음.
- 도입부에서 낮은 기대치나 평범한 첫인상을 먼저 말한 뒤, 그것이 어떻게 달라졌는지로 이어간다 (반전 구조).

### 5-3. 본문 구조
- 소제목은 `■` 기호 + 질문형 또는 화제 제시형 문구.
  (예: "■ 우리가 놓치고 있는 진짜 변화", "■ 처음 2주 — 기대 이상이었던 것들")
- 소제목 안에 종종 "왜 ~일까요", "~라면" 같은 질문 톤을 넣는다.
- 섹션 사이는 `---` 구분선으로 명확히 나눈다.
- 각 섹션은 기승전결이 있는 미니 에피소드처럼 구성 (상황 제시 → 구체적 경험/디테일 → 소감).
- 리스트가 필요할 땐 "첫 번째 / 두 번째 / 세 번째"처럼 자연어 서수로 나열 (번호 매기기 기호 -, 1. 등은 쓰지 않음).

### 5-4. 마무리 패턴
- 글 전체를 관통하는 **한 줄 통찰/격언 style 문장**으로 마무리.
  (예: "그것이 앞으로 10년을 준비하는 가장 확실한 방법입니다.", "단점을 알고 사면 실망 없는 제품.")
- 종종 독자에게 직접 권유/질문을 던지며 끝맺는다.
  (예: "헤드폰 교체를 고민 중이시라면, 한 번 직접 매장에서 써보시길 권해 드립니다.", "다음에 또 가고 싶은 도시가 있냐고 물으신다면, 주저 없이 교토라고 말할 것입니다.")
- 마지막 문단 앞에 종종 "마지막으로 하나만 덧붙이겠습니다" 같은 예고 문구를 쓰기도 함.
- 글 맨 끝은 다시 구분선으로 닫는다:
```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## 6. 그 외 눈에 띄는 특징

1. **정직함을 신뢰의 근거로 삼는다.** 장점만 나열하지 않고 반드시 단점/한계/아쉬운 점을 별도 섹션으로 정직하게 인정한 뒤 결론으로 간다. (스펙 자랑이 아니라 "써본 사람의 리얼함"을 강조)
2. **감각적 디테일을 구체적으로 묘사한다.** 추상적 형용사보다 장면(대사, 소리, 냄새, 행동)으로 보여준다. ("두부가게 할머니가 웃으면서 유부를 하나 건네주셨습니다")
3. **선언 → 반전** 구조를 즐겨 쓴다. 통념이나 예상을 먼저 제시한 뒤 "하지만/그런데 지금은 다릅니다"로 뒤집는다.
4. **숫자를 근거로 쓰되 과장하지 않는다.** ("2~5배 빨라졌습니다", "6개월간 매일 쓴") — 체감/경험 기반의 숫자를 자주 인용.
5. **반말이나 지나친 유행어는 쓰지 않는다.** 항상 존댓말 기본 틀 유지. 다만 구어체 접속사("그런데", "그래서")는 적극 사용해 딱딱함을 없앤다.
6. 이모지, 해시태그, 느낌표 남발은 하지 않는다. 문장부호는 마침표, 쉼표, 물음표, 대시(—) 위주로 절제해서 사용.

## 7. 다른 필자가 이 톤을 따라 쓸 때 체크리스트

- [ ] 첫 문장을 "솔직히" 또는 자기 경험/성향 고백으로 시작했는가?
- [ ] `~습니다` 단정문 사이사이에 `~는데요/인데요`로 숨을 고르는 문장이 있는가?
- [ ] 소제목에 `■`를 쓰고, 질문형 또는 화두형 문구로 달았는가?
- [ ] 섹션 구분에 `---`를 사용했는가?
- [ ] 장점만이 아니라 아쉬운 점/한계도 최소 한 섹션 다뤘는가?
- [ ] 대비 구조("A가 아니라 B", "예전엔 ~였는데 지금은 ~") 를 한 번 이상 썼는가?
- [ ] 짧은 한 문장짜리 강조 문단이 최소 1개 있는가?
- [ ] 마지막 문단이 통찰/격언형 한 줄로 끝나고, 독자에게 말을 거는 문장(권유/질문)으로 마무리되는가?
- [ ] 제목이 `[카테고리] 제목 — 부제` 형식이고, 위아래 `━━━` 구분선으로 감쌌는가?
- [ ] 이모지·과한 느낌표·반말이 섞이지 않았는가?

## 8. 예문으로 보는 Before / After (같은 내용, 톤만 교정)

**Before (밋밋한 정보 전달체)**
> AI 기술은 빠르게 발전하고 있다. 전문가들도 이제 AI를 활용해야 한다.

**After (이 필자의 톤)**
> 솔직히 말씀드리면, 처음엔 저도 "그냥 잘 쓰는 도구네" 정도로만 생각했는데요.
> 그런데 시간이 지나면서 알게 됐습니다 — 문제는 AI를 얼마나 잘 쓰느냐가 아니라, 안 쓰는 게 점점 더 어색해진다는 것입니다.

이처럼 **정보를 먼저 제시하지 않고, 경험 → 반전 → 담담한 통찰**의 순서로 재구성하는 것이 이 문체의 핵심입니다.
